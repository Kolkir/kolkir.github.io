x64 issue solved

Date Wed 15 February 2012

Regarding my previous post I've found answer on StackOverflow. There is some issue in the Visual Studio 2010 x64 C++ compiler - it can't handle object copy in effective way.

MSVS x64 compiler produce ineffective binary copy code, which is two time slower then manual object copy:

#include "timer.h"
#include <vector>
#include <iostream>
#include <algorithm>

class V
{
public:
    double x;
    double y;
    double z;
};

void Test1(std::vector<V>& m)
{
    std::for_each(m.begin(), m.end(),
        [&](V& v)
    {
        v.x = 3.4;
        v.y = 4.66;
        v.z = 56.7;
    });
}

void Test2(std::vector<V>& m)
{
    std::for_each(m.begin(), m.end(),
        [&](V& v)
    {
        V v2;
        v2.x = 3.4;
        v2.y = 4.66;
        v2.z = 56.7;
        v = v2;
    });
}

int main()
{
    cpptask::Timer timer;
    std::vector<V> m(5000000);
    timer.Start();
    Test1(m);
    std::cout << timer.End() << std::endl;
    timer.Start();
    Test2(m);
    std::cout << timer.End() << std::endl;

    return 0;
}

But if i add V::operator= times became the same. After analyzing generated assemblies I saw that x64 compiler use SSEx instruction by default in all cases, and in this case it cause time overhead.
